## 数据库概述

### 数据库架构

```
水面漂浮物检测系统数据库 (yyxz_sqlite.db)
├── user (用户表)
├── datasets (数据集表)
├── models (模型表)
├── detection_records (图片检测记录表)
├── detection_crops (检测目标裁剪图表)
├── video_detection_records (视频检测记录表)
├── video_detection_frames (视频检测关键帧表)
├── evaluation_records (模型评估记录表)
├── realtime_detection_sessions (实时检测会话表)
└── realtime_detection_detections (实时检测记录表)
```

### 设计原则

1. **主键设计**: 用户表使用自增ID，其他表使用UUID字符串
2. **外键约束**: 使用外键保证数据完整性和引用一致性
3. **时间戳**: 所有表包含创建时间字段，部分表包含更新/开始/完成时间
4. **状态管理**: 使用状态字段标识数据的不同生命周期阶段
5. **文件存储**: 采用bucket+object_key模式存储文件路径

## 数据表设计

### 1. 用户表 (user)

**表名**: `user`
**说明**: 存储系统用户的基本信息和权限设置

#### 表结构

| 字段名 | 数据类型 | 长度 | 主键 | 非空 | 唯一 | 默认值 | 说明 |
|--------|----------|------|------|------|------|--------|------|
| id | INTEGER | - | ✓ | ✓ | - | AUTOINCREMENT | 用户ID，自增主键 |
| username | VARCHAR | 50 | - | ✓ | ✓ | - | 用户名，登录账号 |
| password | VARCHAR | 100 | - | ✓ | - | - | 密码（MD5加密） |
| real_name | VARCHAR | 50 | - | - | - | NULL | 真实姓名 |
| phone | VARCHAR | 20 | - | - | - | NULL | 联系电话 |
| email | VARCHAR | 100 | - | - | - | NULL | 电子邮箱 |
| avatar_bucket | VARCHAR | 50 | - | - | - | NULL | 头像存储桶 |
| avatar_object_key | VARCHAR | 255 | - | - | - | NULL | 头像对象键 |
| role | INTEGER | - | - | ✓ | - | 0 | 用户角色：0-普通用户，1-管理员 |
| status | INTEGER | - | - | ✓ | - | 1 | 用户状态：0-禁用，1-启用 |
| create_time | TIMESTAMP | - | - | ✓ | - | CURRENT_TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | - | - | ✓ | - | CURRENT_TIMESTAMP | 更新时间 |

#### 字段说明

- **username**: 用户登录账号，系统唯一标识
- **password**: 使用MD5加密存储的密码
- **role**: 角色控制，区分管理员(1)和普通用户(0)的权限
- **status**: 账号状态控制，禁用(0)的账号无法登录
- **avatar_bucket/avatar_object_key**: 采用对象存储模式管理用户头像

#### 索引

- **PRIMARY KEY**: `id`
- **UNIQUE INDEX**: `username`

### 2. 数据集表 (datasets)

**表名**: `datasets`
**说明**: 管理水面漂浮物检测数据集

#### 表结构

| 字段名 | 数据类型 | 长度 | 主键 | 非空 | 唯一 | 默认值 | 说明 |
|--------|----------|------|------|------|------|--------|------|
| id | TEXT | - | ✓ | ✓ | - | - | 数据集ID（UUID） |
| name | TEXT | - | - | ✓ | - | - | 数据集名称 |
| description | TEXT | - | - | - | - | NULL | 数据集描述 |
| zip_bucket | TEXT | - | - | ✓ | - | - | ZIP文件存储桶 |
| zip_object_key | TEXT | - | - | ✓ | - | - | ZIP文件对象键 |
| structure_info | TEXT | - | - | - | - | NULL | 数据集结构信息(JSON格式) |
| upload_time | TEXT | - | - | ✓ | - | - | 上传时间 |
| file_size | INTEGER | - | - | - | - | NULL | 文件大小(字节) |
| status | INTEGER | - | - | - | - | 0 | 验证状态：0-待验证，1-验证通过，2-验证失败 |

#### 字段说明

- **id**: 使用UUID作为主键，全局唯一

  ```json
  {
    "total_files": 5544,
    "train_images": 4032,
    "train_labels": 4032,
    "valid_images": 907,
    "valid_labels": 907,
    "test_images": 605,
    "test_labels": 605
  }
  ```
- **status**: 数据集验证状态，只有验证通过的数据集才能用于训练

- **PRIMARY KEY**: `id`

| chinese_name | TEXT | - | - | ✓ | - | - | 检测类别（中文） |
| confidence | REAL | - | - | ✓ | - | - | 置信度分数 |
| bbox | TEXT | - | - | ✓ | - | - | 边界框坐标(JSON格式) |
| detection_result | TEXT | - | - | - | - | NULL | 完整检测结果(JSON格式) |
| detection_time | TEXT | - | - | ✓ | - | - | 检测时间 |
| cropped_image_bucket | TEXT | - | - | ✓ | - | - | 目标裁剪图存储桶 |
| cropped_image_object_key | TEXT | - | - | ✓ | - | - | 目标裁剪图对象键 |
| create_time | TEXT | - | - | ✓ | - | - | 创建时间 |

#### 字段说明

- **session_id**: 外键，关联realtime_detection_sessions表的id字段，支持级联删除
- **frame_number**: 该检测发生时的帧序号
- **track_id**: ByteTrack算法分配的目标跟踪ID，用于追踪同一目标
- **class_name**: 检测类别的英文名称（如 floating_object）
- **chinese_name**: 检测类别的中文名称（如 漂浮物）
- **bbox**: JSON格式的边界框坐标 [x1, y1, x2, y2]
- **frame_result_image_bucket/frame_result_image_object_key**: 带检测框的帧结果图存储位置
- **cropped_image_bucket/cropped_image_object_key**: 检测目标裁剪图的存储位置

#### 外键约束

- **FOREIGN KEY**: `session_id` REFERENCES `realtime_detection_sessions(id)` ON DELETE CASCADE

#### 索引

- **PRIMARY KEY**: `id`
- **INDEX**: `session_id` (idx_rt_detections_session)
- **INDEX**: `session_id, track_id, class_name` (idx_rt_detections_track)
- **INDEX**: `session_id, frame_number` (idx_rt_detections_frame)

## 数据表关系

1. **user ← detection_records**: 一个用户可以有多条图片检测记录
2. **user ← video_detection_records**: 一个用户可以有多条视频检测记录
3. **user ← realtime_detection_sessions**: 一个用户可以有多个实时检测会话
4. **user ← evaluation_records**: 一个用户可以有多条模型评估记录
5. **datasets ← models**: 一个数据集可以用于训练多个模型
6. **models ← detection_records**: 一个模型可以被用于多次图片检测
7. **models ← video_detection_records**: 一个模型可以被用于多次视频检测
8. **models ← realtime_detection_sessions**: 一个模型可以被用于多个实时检测会话
9. **models ← evaluation_records**: 一个模型可以被评估多次
10. **detection_records ← detection_crops**: 一个检测记录可以包含多个目标裁剪图
11. **video_detection_records ← video_detection_frames**: 一个视频检测记录可以包含多个关键帧
12. **realtime_detection_sessions ← realtime_detection_detections**: 一个实时检测会话可以包含多条检测记录

## 索引设计

### 主键索引

所有表都有主键索引，确保数据唯一性和快速查询。

### 外键索引

为提高关联查询性能，对所有外键字段建立索引：

```sql
-- user_id 索引
CREATE INDEX idx_detection_records_user_id ON detection_records(user_id);
CREATE INDEX idx_video_detection_records_user_id ON video_detection_records(user_id);
CREATE INDEX idx_evaluation_records_user_id ON evaluation_records(user_id);

-- model_id 索引
CREATE INDEX idx_detection_records_model_id ON detection_records(model_id);
CREATE INDEX idx_video_detection_records_model_id ON video_detection_records(model_id);
CREATE INDEX idx_evaluation_records_model_id ON evaluation_records(model_id);

-- dataset_id 索引
CREATE INDEX idx_models_dataset_id ON models(dataset_id);

-- record_id 索引 (detection_crops, video_detection_frames)
CREATE INDEX idx_detection_crops_record_id ON detection_crops(record_id);
CREATE INDEX idx_video_detection_frames_record_id ON video_detection_frames(record_id);

-- 实时检测索引
CREATE INDEX idx_rt_sessions_user ON realtime_detection_sessions(user_id);
CREATE INDEX idx_rt_detections_session ON realtime_detection_detections(session_id);
CREATE INDEX idx_rt_detections_track ON realtime_detection_detections(session_id, track_id, class_name);
CREATE INDEX idx_rt_detections_frame ON realtime_detection_detections(session_id, frame_number);
```

### 业务索引

根据业务查询需求建立的索引：

```sql
-- 按时间查询的索引
CREATE INDEX idx_detection_records_create_time ON detection_records(create_time);
CREATE INDEX idx_video_detection_records_create_time ON video_detection_records(create_time);
CREATE INDEX idx_evaluation_records_create_time ON evaluation_records(create_time);

-- 视频检测状态索引
CREATE INDEX idx_video_detection_records_status ON video_detection_records(status);

-- 模型状态索引
CREATE INDEX idx_models_status ON models(status);
```

## 建表SQL语句

### 1. 用户表

```sql
CREATE TABLE IF NOT EXISTS user (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    real_name VARCHAR(50),
    phone VARCHAR(20),
    email VARCHAR(100),
    avatar_bucket VARCHAR(50),
    avatar_object_key VARCHAR(255),
    role INTEGER NOT NULL DEFAULT 0,
    status INTEGER NOT NULL DEFAULT 1,
    create_time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 2. 数据集表

```sql
CREATE TABLE IF NOT EXISTS datasets (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    zip_bucket TEXT NOT NULL,
    zip_object_key TEXT NOT NULL,
    structure_info TEXT,
    upload_time TEXT NOT NULL,
    file_size INTEGER,
    status INTEGER DEFAULT 0
);
```

### 3. 模型表

```sql
CREATE TABLE IF NOT EXISTS models (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    dataset_id TEXT,
    base_model TEXT,
    status SMALLINT DEFAULT 0,
    model_bucket TEXT,
    model_object_key TEXT,
    train_images TEXT,
    train_files TEXT,
    create_time TEXT NOT NULL,
    FOREIGN KEY (dataset_id) REFERENCES datasets (id)
);
```

### 4. 图片检测记录表

```sql
CREATE TABLE IF NOT EXISTS detection_records (
    id TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    model_id TEXT NOT NULL,
    original_image_bucket TEXT NOT NULL,
    original_image_object_key TEXT NOT NULL,
    result_image_bucket TEXT,
    result_image_object_key TEXT,
    detection_result TEXT,
    confidence_threshold REAL DEFAULT 0.5,
    create_time TEXT NOT NULL,
    title TEXT,
    description TEXT,
    enhanced_image_bucket TEXT,
    enhanced_image_object_key TEXT,
    analysis_result TEXT,
    suggestions TEXT,
    FOREIGN KEY (user_id) REFERENCES user (id),
    FOREIGN KEY (model_id) REFERENCES models (id)
);
```

### 5. 检测目标裁剪图表

```sql
CREATE TABLE IF NOT EXISTS detection_crops (
    id TEXT PRIMARY KEY,
    record_id TEXT NOT NULL,
    object_index INTEGER NOT NULL,
    class_name TEXT NOT NULL,
    chinese_name TEXT NOT NULL,
    confidence REAL NOT NULL,
    bbox TEXT NOT NULL,
    image_bucket TEXT NOT NULL,
    image_object_key TEXT NOT NULL,
    create_time TEXT NOT NULL,
    FOREIGN KEY (record_id) REFERENCES detection_records (id)
);
```

### 6. 视频检测记录表

```sql
CREATE TABLE IF NOT EXISTS video_detection_records (
    id TEXT PRIMARY KEY,
    user_id INTEGER NOT NULL,
    model_id TEXT NOT NULL,
    original_video_bucket TEXT NOT NULL,
    original_video_object_key TEXT NOT NULL,
    result_video_bucket TEXT,
    result_video_object_key TEXT,
    detection_result TEXT,
    confidence_threshold REAL DEFAULT 0.5,
    frame_rate INTEGER DEFAULT 30,
    total_frames INTEGER,
    processed_frames INTEGER,
    target_frames INTEGER,
    actual_processed_frames INTEGER DEFAULT 0,
    create_time TEXT NOT NULL,
    start_time TEXT,
    complete_time TEXT,
    status INTEGER DEFAULT 0,
    title TEXT,
    description TEXT,
    FOREIGN KEY (user_id) REFERENCES user (id),
    FOREIGN KEY (model_id) REFERENCES models (id)
);
```

### 7. 视频检测关键帧表

```sql
CREATE TABLE IF NOT EXISTS video_detection_frames (
```

