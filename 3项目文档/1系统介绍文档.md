# 基于「YOLO目标检测 + 多模态AI分析」的水面漂浮物检测分析系统

## 项目概述

本项目是一个基于人工智能技术的水面漂浮物检测系统，旨在通过先进的计算机视觉技术和深度学习模型，为水环境监测与治理提供智能化的漂浮物识别服务。系统集成了图片检测、视频检测、实时检测、模型评估、模型管理、数据集管理等多项功能，构建了一个完整的水面漂浮物检测服务平台。

### 核心创新点

1. **漂浮物精准检测**: 支持水面漂浮物的智能识别和定位
2. **多模态检测方式**: 支持图片、视频、实时摄像头三种检测模式
3. **YOLO模型集成**: 集成YOLOv8/YOLO11/YOLO12/YOLO26等先进目标检测模型
4. **智能检测分析**: 水面漂浮物自动识别和定位
5. **分层权限管理**: 支持管理员和普通用户的差异化功能访问
6. **完整工作流程**: 从数据集管理到模型管理再到漂浮物检测的完整闭环
7. **多模态大模型辅助分析**: 使用多模态大模型根据YOLO检测识别结果和图片进一步分析并给出合理治理建议(属于图片识别功能模块)
8. **模型评估系统**: 支持GT标签与预测结果对比，自动计算精度指标(Precision/Recall/mAP50)，生成可视化对比图，支持用户评分和筛选

### 应用场景

- **水环境监测**: 湖泊、河流、水库等水域的漂浮物智能监测
- **环保执法**: 水面垃圾污染的自动巡检与取证
- **水务管理**: 水域清洁度评估与清理调度
- **教育培训**: 水环境保护技术和AI检测的教学演示

- **用户评分**: 支持用户对评估结果进行1-5星主观评分和评语

#### 评估指标
- **Precision (精确率)**: 预测为正样本中真正为正样本的比例
- **Recall (召回率)**: 真实正样本中被正确预测的比例
- **mAP50**: IoU阈值为0.5时的平均精度

#### 评估流程
1. 上传待评估图片和对应的YOLO格式标签文件
2. 选择评估模型并设置置信度阈值
3. 系统进行模型推理并计算精度指标
4. 生成GT框与预测框的对比可视化图
5. 用户可对评估结果进行评分和评语
6. 保存评估记录便于后续查看和对比

user = db.query(username, MD5(password))
if not user or user.disabled: return error
token = JWT.encode({ user_id, role, exp: now+24h })
return success(token)

# 鉴权装饰器
@token_required:
    payload = JWT.decode(request.headers.Authorization)
    g.user_id, g.role = payload.user_id, payload.role

# 数据隔离（普通用户只查自己的记录）
if not is_admin():
    conditions += "user_id = g.user_id"
```

### 2. 图片检测完整流程

图片检测包含四个串联步骤：**YOLO推理 → CLAHE图像增强 → Qwen-VL多模态分析 → 治理建议生成**。

# Step1 YOLO检测
image = preprocess(upload)                         # RGBA→RGB
results = YOLO(model).predict(image, conf=threshold)
for box in results.boxes:
    draw_rect_and_chinese_label(image, box, YOLO_CLASSES_CONFIG)
    # 裁剪检测目标并存储
    crop_image = extract_with_padding(image, box.xyxy, padding=0.1)

    db.insert(detection_crops, record_id, object_index, class_name, bbox, crop_key)
upload(original_image, result_image)

lab = BGR→LAB(image)
lab[L] = CLAHE(clipLimit=3.0).apply(lab[L])        # 仅增强亮度通道

# Step3 多模态分析
QwenVL.chat(image_base64, detections_text,

QwenVL.chat(analysis_result,
    prompt="从立即清理/工艺优化/预防策略/长期规划/环境效益五个维度给出建议")
```

### 3. 视频检测流程

视频检测采用**多进程异步架构**：主进程快速返回任务ID，子进程后台逐帧推理，前端轮询进度接口获取百分比。支持帧率采样（跳帧）和自适应MP4编码器选择（avc1 → mp4v → H264）。

```
# 主进程：创建任务，启动子进程后立即返回
record_id = db.insert(status=PENDING)
Process(target=video_process, args=(record_id, ...)).start()
return success(record_id)

# 子进程：逐帧推理
frame_interval = max(1, original_fps / target_fps)  # 帧率采样间隔
while frame = cap.read():
    if frame_idx % frame_interval == 0:
        out.write(YOLO(frame))                       # 推理并写入结果视频
    if frame_idx % 10 == 0:
        db.update(record_id, progress=frame_idx/total)  # 每10帧更新进度

upload(result_video) → db.update(status=COMPLETED)

# 进度查询（前端轮询）
return { progress: processed/total * 100, status }
```

### 4. 实时检测流程

前端每 200ms 将摄像头帧以 Base64 上传，后端使用 **ByteTrack** 跟踪检测，结果图以 Base64 直接返回渲染。对同一会话内**首次出现的 track_id + class_name 组合**才写入数据库，实现去重持久化。

```
# 模型全局缓存（懒加载，避免重复加载）
if model_id not in yolo_models:
    yolo_models[model_id] = YOLO(download_to_cache(model_id))

# 每帧处理
image = base64_decode(request.image_data)
results = model.track(image, persist=True, tracker="bytetrack.yaml")

for box in results.boxes:
    draw_rect_with_track_id(image, box)
    # 新目标去重入库
    if not db.exists(session_id, track_id, class_name):
        file_store.upload(image_screenshot)
        db.insert(realtime_detection_records, ...)

return { result_image: base64(image), new_detections }
```

### 7. 模型评估流程

通过上传图片和YOLO格式标签文件，系统自动对比**GT真值框**与**模型预测框**，计算精度指标并生成可视化对比图。

```
# Step1 解析YOLO标签文件
label_lines = label_file.read().strip().split('\n')
gt_boxes = []
for line in label_lines:
    class_id, x_center, y_center, width, height = line.split()
    # 归一化坐标 → 像素坐标
    x1, y1, x2, y2 = yolo_to_xyxy(x_center, y_center, width, height, img_w, img_h)
    gt_boxes.append({ class_id, bbox: [x1, y1, x2, y2] })

# Step2 模型预测
results = YOLO(model).predict(image, conf=threshold)
pred_boxes = [{ class_id, bbox: box.xyxy, confidence: box.conf } for box in results.boxes]

# Step3 计算IoU并匹配GT框与预测框
matches = []
for gt in gt_boxes:
    best_iou = 0
    best_pred = None
    for pred in pred_boxes:
        if pred.class_id == gt.class_id:
            iou = calc_iou(gt.bbox, pred.bbox)
            if iou > best_iou:
                best_iou = iou
                best_pred = pred
    if best_iou > 0.5:  # IoU阈值
        matches.append({ gt, pred: best_pred, iou: best_iou })

# Step4 计算精度指标
precision = len(matches) / len(pred_boxes) if pred_boxes else 0
recall = len(matches) / len(gt_boxes) if gt_boxes else 0
avg_iou = mean([m.iou for m in matches]) if matches else 0
mAP50 = calculate_mAP(matches, iou_threshold=0.5)

# Step5 绘制对比图（GT框=绿色，预测框=红色）
for gt in gt_boxes:
    draw_box(image, gt.bbox, color=GREEN, label="GT")
for pred in pred_boxes:
    draw_box(image, pred.bbox, color=RED, label=f"{pred.class_name} {pred.confidence:.2f}")
upload(compare_image)

# Step6 保存评估记录
db.insert(evaluation_records, {
    origin_image, compare_image, label_content,
    detection_result: json(pred_boxes),
    metrics: json({ precision, recall, mAP50, avg_iou }),
    score: user_score,  # 用户1-5星评分
    comment: user_comment
})
```

- **认证方式**：除登录/注册外，所有接口均需在请求头携带 `Authorization: Bearer <token>`
- **统一响应格式**：`{ "code": 200, "msg": "success", "data": { ... } }`
- **权限标注**：`公开` 无需登录；`登录` 任意已登录用户；`管理员` 仅 role=1 可访问
