# SYSTEM_OPTIMIZATION_PLAN.md

## 1. 优化目标

当前项目优化不是推倒重来，而是在既有平台基础上提升工程结构清晰度、模块边界稳定性、前后端契约一致性、AI 链路可靠性、视频/实时检测性能与稳定性、文档/测试/协作可维护性。

## 2. 优化方向总览

| 方向 | 优先级 | 目标 |
|---|---:|---|
| 文档与契约体系 | P0 | 让多 Agent 有共同事实来源 |
| 源码结构确认 | P0 | 解决当前源码缺失/待确认边界 |
| API/DB/AI 输出契约 | P0 | 防止跨模块协作破坏兼容性 |
| UI 统一 | P1 | 提升系统展示和使用体验 |
| 视频检测流程 | P1 | 提升长任务可靠性和可观测性 |
| 实时检测性能 | P1 | 降低卡顿、资源占用和重复记录 |
| 模型管理工程化 | P1 | 模型发布、选择、评估更稳定 |
| 测试与回归 | P2 | 支撑持续优化和多 Agent 协作 |
| 扩展能力 | P3 | 多类别、告警、网络摄像头等长期方向 |

## 3. 工程结构优化

当前问题：文档描述的前后端结构与当前工作区可见源码不一致；模块边界、共享契约、测试入口需要沉淀；多 Agent 协作规则刚建立，需要落到任务流程。

建议：补齐源码后先做目录地图，不立即重构；后端按 routes/services/dao/algo/utils 分层梳理但逐步推进；前端按 api/views/components/stores/types/utils/router 梳理；根目录保留工程规划文档作为长期维护入口。

## 4. 前端优化

| 优化项 | 优先级 | 说明 |
|---|---:|---|
| 页面结构清单 | P0 | 先确认实际页面和路由 |
| API 调用统一封装 | P1 | 统一错误处理、loading、鉴权失效 |
| UI 风格统一 | P1 | Element Plus 主题、表格、表单、卡片、状态标签 |
| 检测结果组件化 | P1 | 图片/视频/实时结果统一展示 |
| 模型评估可视化 | P2 | 指标图、对比图、评分筛选 |
| 大屏化 | P3 | 未来展示增强，非当前核心 |

## 5. 后端优化

| 优化项 | 优先级 | 说明 |
|---|---:|---|
| API 清单与返回结构 | P0 | 与前端协作基础 |
| DB 字段和实际代码对齐 | P0 | 防止文档/实现偏差 |
| 视频任务状态机 | P1 | pending/processing/completed/failed/cancelled |
| 文件存储规范 | P1 | 原图、裁剪图、视频、报告、权重路径统一 |
| 权限边界 | P1 | 管理员功能和用户数据隔离 |
| 错误码与日志 | P2 | 便于排查和测试 |

## 6. AI 链路优化

| 优化项 | 优先级 | 说明 |
|---|---:|---|
| 检测结果 schema | P0 | 前端/后端/AI/报告共享 |
| Qwen-VL 分析字段 | P0 | 统一污染描述、风险、治理建议格式 |
| 模型加载缓存 | P1 | 避免重复加载造成延迟 |
| 实时检测跳帧/降采样 | P1 | 平衡 FPS 与精度 |
| 视频关键帧筛选 | P1 | 减少重复帧，提高可读性 |
| 评估脚本化 | P2 | 支撑模型发布前验证 |
| 多类别扩展 | P3 | 需要数据集和 UI/DB/API 协同 |

## 7. 数据库与文件存储优化

当前不重构数据库。建议先做字段字典与实际表结构对齐、索引使用说明、检测记录/关键帧/裁剪图/报告文件生命周期策略、模型权重和训练产物命名规范、数据备份和测试数据清理说明。

## 8. 测试与验收优化

| 测试类型 | 输入 | 验收 |
|---|---|---|
| 图片检测冒烟 | 测试图片 | 返回目标框/标注图/记录 |
| 视频检测冒烟 | 测试视频 | 任务完成、进度合理、关键帧存在 |
| 模型评估 | 评估图片 + 标签 | 指标计算成功、可视化图存在 |
| 实时检测 | 本地 USB 摄像头 | 会话可开始/停止、资源释放 |
| 权限测试 | admin/test | 管理功能权限正确 |
| 报告导出 | 图片检测记录 | Word 文件可生成 |

## 9. 风险清单

| 风险 | 等级 | 应对 |
|---|---:|---|
| 前后端源码未补齐 | 高 | 所有源码级结论标注待确认 |
| 共享契约未固化就并行开发 | 高 | 先写 API/DB/AI schema |
| 模型权重误删/误替换 | 高 | 遵守 AGENTS AI Safety Rules |
| 实时检测资源未释放 | 中 | 增加启动/停止/异常释放验证 |
| LLM API 失败 | 中 | 增加超时、重试、降级提示 |
| UI 优化范围膨胀 | 中 | 先统一现有页面，不做全新大屏 |
| 数据库变更破坏历史数据 | 高 | 当前阶段不重构 DB，后续需迁移方案 |

## 10. 推荐近期路线

完成根目录规划文档评审 → 补齐完整 `web-vue` / `web-flask` 源码 → 做“文档 vs 源码”差异审计 → 固化 API、DB、AI 输出、文件存储契约 → 建立可运行基线和测试 checklist → 分 Agent 小步执行 P1 优化任务。
