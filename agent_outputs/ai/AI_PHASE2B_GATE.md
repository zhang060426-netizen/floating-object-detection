# AI Phase 2B Gate

阶段：Phase 2A 系统契约与重建基线  
角色：AI Agent  
用途：定义 AI 链路进入 Phase 2B 最小代码重建/可运行基线前的门禁。

证据等级统一使用：

- 已源码确认：有 `train.py` / `val.py` / `predict.py` 等源码证据。
- 已资源确认：有 `data.yaml`、weights、output、测试包等资源证据。
- 历史输出确认：来自 `output/` 中历史结果、指标文件。
- 文档推断：来自训练文档、系统文档、规划文档。
- 待后端源码确认：必须等 `web-flask` 推理调用链补齐后确认。
- 冲突/差异：脚本、历史输出、文档之间存在不一致。

| Qwen-VL 契约 | `QWEN_VL_ANALYSIS_SCHEMA.md` 已完成，标注待后端源码确认项 | 已完成本文档配套输出 | 文档推断 |
| metrics 契约 | `EVALUATION_METRICS_SCHEMA.md` 已完成，历史指标和候选字段分离 | 已完成本文档配套输出 | 历史输出确认 + 文档推断 |
| 模型资产基线 | `MODEL_ASSET_BASELINE.md` 已完成，明确基础权重和 `best.pt` 缺失风险 | 已完成本文档配套输出 | 已资源确认 |

## 2. 权重门禁

| 门禁项 | PASS 标准 | BLOCKED 条件 | 当前证据等级 |
|---|---|---|---|
| 基础权重存在 | `weights/yolov8n.pt`、`yolo11n.pt`、`yolo12n.pt`、`yolo26n.pt` 均存在 | 基础权重缺失或被替换未记录 | 已资源确认 |
| 已训练权重状态明确 | 若 `best.pt` 缺失，必须明确降级或阻塞 | 宣称 `best.pt` 存在但文件系统未确认 | 已资源确认 |
| 权重不可变更 | Phase 2A/2B Gate 前不得替换、删除、移动权重 | 任意权重文件被修改 | 文档推断 |
| 发布权重来源 | 后端确认模型管理上传/发布逻辑后才能进入真实推理基线 | 后端源码缺失导致来源不明 | 待后端源码确认 |
| 历史精度声明 | 只能声明历史输出指标，不能声明当前权重可复现该精度 | 用基础权重冒充历史训练权重 | 历史输出确认 + 冲突/差异 |

## 3. 类别定义门禁

| 门禁项 | PASS 标准 | BLOCKED 条件 | 当前证据等级 |
|---|---|---|---|
| 类别数量 | `nc: 1` 保持不变 | 修改 `nc` 或引入多类别 | 已资源确认 |
| 类别名称 | `names: ['floating_object']` 保持不变 | 修改类别名导致前端/报告/评估错位 | 已资源确认 |
| 标签类别 ID | 当前测试/训练资源类别 ID 为 `0` | 出现未登记类别 ID 且无迁移方案 | 已资源确认 |
| 前后端同步 | 后端/前端/报告使用同一 class map | 后端源码未确认前不得声明已同步 | 待后端源码确认 |
| 扩展策略 | 多类别只能进入后续受控扩展阶段 | Phase 2B 直接改类别 | 文档推断 |

## 4. 推理输出 schema 门禁

| 门禁项 | PASS 标准 | BLOCKED 条件 | 当前证据等级 |
|---|---|---|---|
| 最小检测字段 | `class_id`、`class_name`、`confidence`、`bbox`、`bbox_format` 明确 | 缺少目标框或类别字段 | 文档推断 |
| 坐标格式 | 明确 `bbox_xyxy` / `bbox_xywhn` 的主次关系和图像尺寸 | 前后端无法判断 bbox 坐标含义 | 文档推断 + 待后端源码确认 |
| summary | 至少定义目标总数和类别统计 | 前端/报告无法展示摘要 | 文档推断 |
| artifacts | 原图、标注图、裁剪图、结果视频、关键帧 key 规则由后端确认 | 文件路径/URL 规则不明 | 待后端源码确认 |
| 版本字段 | `schema_version` 必须存在 | schema 无版本，后续兼容风险高 | 文档推断 |
| 后端 review | Backend 确认可落库/可返回/可供前端消费 | 后端源码缺失或未 review | 待后端源码确认 |

## 5. Qwen-VL schema 门禁

| 门禁项 | PASS 标准 | BLOCKED 条件 | 当前证据等级 |
|---|---|---|---|
| 输入上下文 | 明确图片引用、YOLO 摘要、prompt 元信息 | Qwen-VL 输入不含检测上下文 | 文档推断 |
| 输出字段 | 污染描述、风险等级、风险原因、治理建议、限制说明均有候选字段 | 输出纯文本无法稳定被前端/报告消费 | 文档推断 |
| raw_response | 明确是否保存原始返回以及脱敏策略 | 保存密钥、Authorization、Base64 大图等敏感信息 | 文档推断 |

| API 不替换 | 保持文档所述阿里云百炼/Qwen-VL 方向，Phase 2A 不替换 | 未经 Leader 决策更换供应商 | 文档推断 |
| 后端配置 | 确认 `web-flask/algo/llm/config.py` 或等价位置 | 当前源码缺失 | 待后端源码确认 |

## 6. evaluation metrics 门禁

| 门禁项 | PASS 标准 | BLOCKED 条件 | 当前证据等级 |
|---|---|---|---|
| 历史指标保留 | 历史验证/测试 P、R、mAP50、mAP50-95 作为历史输出记录 | 将历史指标改写为当前可复现结果 | 历史输出确认 |
| overall 字段 | `precision`、`recall`、`map50`、`map50_95` 明确 | 指标命名不统一 | 历史输出确认 |
| per_class 字段 | 单类别 `floating_object` 指标结构明确 | 多类别结构缺失导致后续扩展困难 | 已资源确认 + 文档推断 |
| IoU/match | GT/预测匹配字段和阈值候选明确 | 无法解释 Precision/Recall 来源 | 文档推断 |
| artifacts | 曲线图、混淆矩阵、对比图字段明确 | 评估结果无法可视化追踪 | 历史输出确认 + 文档推断 |
| 后端评估实现 | Backend 确认 `/api/evaluation`、IoU 计算、metrics JSON | 当前后端源码缺失 | 待后端源码确认 |

## 8. 阻塞条件

| 阻塞项 | 阻塞原因 | 解除条件 | 证据等级 |
|---|---|---|---|
| 后端源码仍缺失 | 无法确认应用内推理调用链 | 补齐 `web-flask` app/routes/services/algo/db | 待后端源码确认 |
| 真实 `best.pt` 缺失且无降级策略 | 无法建立可信推理基线 | 补齐权重或 Leader 批准开发占位策略 | 已资源确认 |
| 类别定义被修改 | 会破坏前端展示、评估、报告和历史指标一致性 | 恢复 `class_id=0/floating_object` 或走正式迁移方案 | 已资源确认 |
| schema 未被 Backend review | AI 输出无法保证可落库/返回 | Backend 出具 `DETECTION_RESULT_SCHEMA` review | 待后端源码确认 |
| Qwen-VL 配置缺失 | 无法确认分析链可运行 | 补齐/确认 LLM config 和调用链 | 待后端源码确认 |
| 评估 API 未确认 | 无法执行评估冒烟 | Backend 确认 `/api/evaluation` 最小实现 | 待后端源码确认 |
| 任意禁止动作发生 | 训练/验证/预测/替换权重/改脚本违反边界 | 回滚变更并重新审计 | 文档推断 |
