# System Rebuild Plan

更新时间：2026-05-13  
阶段：Phase 2A：系统契约与重建基线  
边界：只写重建计划；禁止补齐 `web-vue` / `web-flask` 源码，禁止修改数据库、模型、测试资源。

## 1. 重建目标

| 目标 | 说明 | 交付物 | 证据等级 |
|---|---|---|---|
| 恢复或明确最小源码范围 | 判断完整前后端源码是否可恢复；不可恢复时定义最小重建边界 | 源码恢复任务与 Gate | 文档推断 |
| 固化契约草案 | API、DB、检测结果、AI 输出、文件存储、JWT、metrics 均有草案 | `agent_outputs/*/*.md` | 文档推断 |
| 建立测试资源映射 | 将图片、视频、评估、实时限制映射到冒烟场景 | `SMOKE_TEST_RESOURCE_MAP.md` | 已资源确认 |
| 进入 Phase 2B 前门禁 | 确保不在源码缺失下直接实现 | `PHASE2B_GATE_CHECKLIST.md` | 文档推断 |

## 2. 当前不可直接进入代码实现的原因

| 原因 | 影响 | 当前证据 | 证据等级 |
|---|---|---|---|
| 前端缺少 `src/` 和运行入口 | 无法构建、运行、确认页面/API | FE Phase 1 审计 | 冲突/差异 |
| 后端缺少 Flask 入口和 routes/services | 无法确认 API、DB、鉴权、推理链 | BE Phase 1 审计 | 冲突/差异 |
| DB 只有文档定义 | 无法确认实际 DB 已创建或字段一致 | DB 文档 + 后端源码缺失 | 数据库文档确认 |
| 应用内 YOLO 推理封装缺失 | 无法确认 detection_result 真实结构 | AI/Backend Phase 1 审计 | 待源码确认 |
| 测试资源存在但系统不可运行 | 禁止端到端冒烟测试 | Phase 2A 边界 | 已资源确认 |

## 12. 回滚与风险控制

| 风险 | 控制方式 | 回滚方式 | 证据等级 |
|---|---|---|---|
| 基于文档猜测直接写代码 | Gate 阻断，要求证据等级标注 | 删除未通过 Gate 的实现分支 | 文档推断 |
| 契约字段与后续源码不一致 | 所有契约标注待源码确认 | 按源码事实修订契约 | 待源码确认 |
| 模型权重被误替换 | Phase 2A 禁止修改权重 | 从原始资源恢复 | 已资源确认 |
| DB 被误改 | Phase 2A 禁止修改数据库 | 恢复 DB 文件或迁移脚本 | 数据库文档确认 |
| 测试资源被移动/删除 | Phase 2A 禁止移动测试资源 | 从原测试包恢复 | 已资源确认 |
