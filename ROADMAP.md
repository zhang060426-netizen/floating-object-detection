# ROADMAP.md

## 1. 路线总原则

当前项目已具备完整平台雏形，后续不应先做大重构，而应按“文档契约 → 可运行基线 → 小步优化 → 多 Agent 并行”的方式推进。

优先级规则：

- **P0**：影响所有 Agent 协作和系统可理解性的基础工作。
- **P1**：当前系统核心体验/稳定性优化。
- **P2**：增强能力和工程质量提升。
- **P3**：未来扩展，不进入近期目标。

## 2. 阶段划分

### Phase 0：规划与契约固化（当前阶段）

目标：让后续 Agent 能直接拆任务执行。

交付：`README.md`、`ARCHITECTURE.md`、`MODULE_BOUNDARIES.md`、`MULTI_AGENT_PLAN.md`、`AI_PIPELINE_ANALYSIS.md`、`SYSTEM_OPTIMIZATION_PLAN.md`、`ROADMAP.md`。

验收：模块边界清楚、目录职责清楚、数据流与 AI 链路清楚、多 Agent 拆分清楚、优先级/依赖/风险清楚、明确推断边界和待源码确认项。

### Phase 1：源码补齐与基线验证

目标：确认文档描述与实际源码一致，建立可运行基线。

| 任务 | Owner | 依赖 | 输出 |
|---|---|---|---|
| 补齐/确认完整 `web-vue` 源码结构 | Frontend Agent | 完整源码 | 页面/路由/API 调用清单 |
| 补齐/确认完整 `web-flask` 源码结构 | Backend Agent | 完整源码 | routes/services/db/algo 清单 |
| 对齐 DB 文档与实际初始化脚本 | Backend + Docs | 后端源码 | 字段差异报告 |
| 固化检测结果 JSON 契约 | Backend + AI + Frontend | 推理源码 | schema 文档 |
| 建立最小启动验证步骤 | Docs/Test Agent | 启动教程 | 启动 checklist |

### Phase 2：核心体验与稳定性优化

| 方向 | 任务 | Owner | 说明 |
|---|---|---|---|
| UI 统一 | 统一布局、菜单、状态提示、检测结果展示 | Frontend | 先统一现有页面，不做大屏重做 |
| 视频检测 | 状态机、进度、失败恢复、关键帧策略 | Backend + AI | 优先稳定性 |
| 实时检测 | 模型缓存、摄像头释放、跳帧策略 | Backend + AI | 优先资源释放和性能基线 |
| 模型管理 | 发布流程、模型可用性校验、指标展示 | Backend + Frontend | 保持兼容 |
| 文档同步 | API/DB/AI/测试文档与源码对齐 | Docs | 每次改动同步 |

### Phase 3：工程质量提升

- 前端 API 类型定义与统一错误处理。
- 后端 routes/services/dao 分层梳理。
- 文件存储路径规范与清理策略。
- 数据库索引和查询优化。
- 自动化模型评估脚本。
- 基于 `4测试包/` 建立回归测试清单。
- Word 报告模板字段稳定。
- LLM 调用超时、降级、日志脱敏。

### Phase 4：能力扩展（暂缓）

多类别检测、GPU 推理优化、实时告警系统、水域治理数据分析、网络摄像头接入、UI 大屏化。暂不做微服务、Kubernetes、云原生、多摄像头集群、商业支付、移动端 App。

## 3. 依赖关系

规划文档体系 → 源码补齐/结构确认 → 共享契约确认 → 可运行基线 → 前端 UI 统一 / 后端流程稳定 / AI 链路性能基线 → 端到端体验优化 → 自动化测试与长期维护。

## 4. 风险与控制

| 风险 | 影响 | 控制方式 |
|---|---|---|
| 前后端源码缺失 | 无法源码级确认 | 标注推断，后续源码补齐后复核 |
| 共享契约不稳定 | 多 Agent 互相破坏 | 先写 schema/API/DB 文档 |
| 视频/实时检测资源泄漏 | 系统卡顿或摄像头占用 | 状态机、超时、释放策略 |
| LLM API 不稳定 | AI 分析失败 | 超时、重试、降级、错误提示 |
| 模型替换影响精度 | 识别效果下降 | 模型发布前必须提供评估结果 |

## 5. 近期推荐执行顺序

1. 完成并评审本轮 7 份规划文档。
2. 补齐完整前后端源码后，由 Documentation Agent 对照文档做差异清单。
3. Backend Agent 固化 API、DB、文件存储、检测结果契约。
4. Frontend Agent 基于契约统一页面和 API 调用。
5. AI Agent 梳理推理/评估链路与性能基线。
6. Documentation/Test Agent 建立启动、测试、回归、验收清单。
